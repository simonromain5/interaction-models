Brownian motion¶

class brownian.BrownianMotion(dt, radius, n_particles, surface, n_steps)¶

This class represents the Brownian motion of particles in 2 dimensions. The motion of the particles is restricted in a box. We consider the particles as ghost particles: they can intermingle.

Parameters:
  • dt (float or int) – Increment of time for each step. Constant * dt is the variance of the normal distribution that we use to calculate the increment of all the positions at each step.
  • radius (float or int) – radius of the particles. It as constant for all the particles
  • n_particles (int) – Number of particles in the box
  • surface (float or int) – Surface of the box. We consider the box as a square, hence the length of the side is equal to the square root of the surface.
  • n_steps (int) – Number of steps that we consider for the total movement of the particles.
brown_iter_2d()¶

Returns an array of the increment of the next position of the particles (dx, dy). As we consider a Brownian motion, the increment follows a 2D gaussian law of mean 0 and of variance dt.

Returns:Returns an array of the increment of the next position of the particles (dx, dy). It is of shape (n_particles, 2)
Return type:np.array

iter_movement(step, animation=False)¶

This function updates the self.position_array at time step*dt. The function takes the position of the array (x, y) and adds a Brownian infinitesimal step (dx, dy). Hence the new position is (x+dx, y+dy). The borders of the box are also considered directly in the function. If after iteration, x+dx or y+dy is inferior to 0 then the position is updated to x+dx=0 or y+dy=0 respectively. If after iteration, x+dx or y+dy is superior to self.side then the position is updated to x+dx=self.side or y+dy=self.side respectively.

Parameters:
  • step (int) – step of the iteration. It ranges from 0 to self.n_steps-1
  • animation (bool, optional) – This parameter is set to False by default. This means that the creation_tij array is stored and can be analyzed. It is set to true only when the animation is run. As the animation can run indefinitely, too much data can be stored
total_movement()¶

This function iterates all the Brownian motion throughout the n_steps and returns the tij array to be analyzed

Returns:Returns the tij array. It represents all the interactions between particles i and j at time t
Return type:np.array
